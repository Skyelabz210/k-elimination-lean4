Deep Dive FAQ

Technical questions answered.

Why does K-Elimination work?

The core insight is the key congruence: X mod A = (X mod M + k·M) mod A. Since X = vₘ + k·M by definition, taking both sides mod A reveals that k·M is encoded in (vₐ - vₘ). Multiplying by M⁻¹ extracts k exactly because k < A.

What are the requirements?

Coprimality: gcd(M, A) = 1 — ensures M⁻¹ exists.
Range bound: X ∈ [0, M·A) — ensures k < A so k mod A = k.

What if X ≥ M·A?

The formula gives k mod A, not k. For extended ranges: use anchor stacking (chain multiple anchors), tier promotion (fall back to arbitrary-precision), or range partitioning (M·A-sized chunks).

How is M⁻¹ computed?

Extended Euclidean Algorithm gives M⁻¹ in O(log A). For fixed A, precompute once — then each division is O(1).

What's the relationship to CRT?

Complementary. CRT reconstructs X from residues; K-Elimination extracts the quotient directly without full reconstruction. For division by M: K-Elimination is O(k) vs CRT's O(k²).

Why is it so fast (26-29 ns)?

Three modular operations: two mod (already have vₐ, vₘ), one multiply, one mod. No loops, no iteration, no reconstruction. Algebraically direct.

How does this enable bootstrap-free FHE?

FHE rescaling divides ciphertext coefficients. Traditional methods need bootstrapping or use floating-point (drift). K-Elimination provides exact integer division — no bootstrapping, no drift.

What is "zero-drift" arithmetic?

In CKKS, rescaling errors compound. K-Elimination's 100% exactness means no drift — the 1000th operation is as precise as the first.

Can it handle signed numbers?

Yes. Use offset encoding: [-N, N) → [0, 2N). Apply K-Elimination, then re-center. Coprimality and range requirements apply to encoded values.

What about multi-level RNS?

For k-level RNS with moduli M₁...Mₖ, apply K-Elimination iteratively or use anchor stacking. The key congruence generalizes to each level.

Connection to Montgomery reduction?

Complementary. Montgomery handles modular multiplication; K-Elimination handles exact division. Montgomery keeps you in residue form for mul; K-Elimination lets you divide without leaving it.
